# HTML Section to Slide Type Mapping

Reference for parsing Snowflake-branded HTML (generated by `snowflake-brand-guidelines` skill) and mapping sections to PowerPoint slide types.

## How Brand HTML is Structured

The `snowflake-brand-guidelines` skill produces HTML with:
- CSS custom properties (`:root { --sf-blue: #29B5E8; ... }`)
- Google Fonts: Montserrat (headings, 800 weight) and Lato (body)
- Consistent class names across all generated documents
- A dark gradient header, content sections, and a copyright footer

## Section Detection Rules

Use BeautifulSoup to parse the HTML. Walk through top-level elements in document order and classify each one.

### 1. Header (Title Slide)

**How to detect:**
```python
header = soup.find(class_="header")
# or: soup.find("header")
# or: element with style containing "linear-gradient"
```

**Content to extract:**
- `h1` text → slide title (apply colorstack split)
- First `p` or `.header-subtitle` → subtitle
- Any keyword badges/pills → pill labels

**Maps to:** Title Slide (slide-types.md #1)

### 2. Intro / Problem Section

**How to detect:**
```python
# First <section> after header that contains mostly <p> text (no cards/grids)
section = soup.find("section")
has_cards = section.find(class_=re.compile(r"pillar|card|grid|stat"))
is_intro = not has_cards and len(section.find_all("p")) >= 1
```

**Content to extract:**
- First `h2` or strong opening sentence → big statement
- Remaining `p` text → body narrative

**Maps to:** Problem/Intro Slide (slide-types.md #2)

### 3. Pillar Grid (Executive Summary)

**How to detect:**
```python
pillars = soup.find(class_=re.compile(r"pillars|summary-grid|card-grid"))
# or: a container with 2-4 child elements that each have h3 + ul
```

**Content to extract per card:**
- `h3` text → pillar name
- `border-top` color or accent class → card color
- First `p` or `strong` → tagline
- `ul > li` items → bullet points

**Color assignment:** Cycle through `[SF_BLUE, SF_STAR_BLUE, SF_MID_BLUE]` for 3 pillars, or `[SF_BLUE, SF_STAR_BLUE, SF_MID_BLUE, SF_ORANGE]` for 4.

**Maps to:** Executive Summary Slide (slide-types.md #3)

**Also creates:** Individual Detail Slides (slide-types.md #4) for each pillar, if the HTML has substantial bullet content per pillar.

### 4. Feature / Detail Section

**How to detect:**
```python
# <section> with an <h2> heading and a <ul> bullet list
section.find("h2") and section.find("ul")
# Not inside a pillar grid
```

**Content to extract:**
- `h2` text → heading (uppercase)
- First `p` → intro line
- `li` items → bullets (limit to 4-5 per slide)

**Icon selection:** Match heading keywords to icons:
- "easy", "simple", "fast", "quick" → `\u26A1`
- "connect", "integrate", "unified", "data" → `\U0001F517`
- "trust", "secure", "govern", "protect" → `\U0001F6E1`
- "scale", "perform", "analytics" → `\U0001F4CA`
- "ai", "intelligent", "ml", "model" → `\U0001F9E0`
- Default → `\u2699`

**Maps to:** Detail Slide (slide-types.md #4)

### 5. Comparison Table

**How to detect:**
```python
table = soup.find("table")
# or: section containing a <table> with <thead> and <tbody>
```

**Content to extract:**
- `thead > tr > th` → column headers
- `tbody > tr > td` → cell values
- Detect indicators: `✓` or `\u2713`, `✗` or `\u2717`, `~`

**Maps to:** Comparison Table Slide (slide-types.md #5)

### 6. Stat Cards

**How to detect:**
```python
stats = soup.find_all(class_=re.compile(r"stat-card|metric|number-card"))
# or: elements with a large number (font-size > 2em) and a label
```

**Content to extract:**
- Large text (number/percentage) → stat value
- Small text / label → stat description

**Maps to:** Stats Slide (slide-types.md #6)

### 7. Callout Box

**How to detect:**
```python
callout = soup.find(class_=re.compile(r"callout|highlight|quote"))
# or: element with colored left border and background
```

**Content to extract:**
- Main text content → callout quote/insight

**Maps to:** Callout Slide (slide-types.md #8) OR fold into an adjacent detail slide as a highlighted text box.

### 8. CTA / Call-to-Action

**How to detect:**
```python
cta = soup.find(class_=re.compile(r"cta|call-to-action"))
# or: section with a button/link containing "Get Started", "Learn More", "Contact"
```

**Content to extract:**
- Heading text → CTA headline
- Button text → button label

**Maps to:** CTA/Closing Slide (slide-types.md #7)

### 9. Footer

**How to detect:**
```python
footer = soup.find(class_="footer")
# or: soup.find("footer")
# or: element containing "\u00A9" copyright symbol
```

**Content to extract:**
- Copyright text → footer line on CTA slide

**Maps to:** Fold into CTA/Closing Slide copyright line.

### 10. Code Blocks

**How to detect:**
```python
code = soup.find_all(["pre", "code"])
```

**Handling:** Generally skip code blocks in presentations. If the code is short and essential (e.g., a 3-line API call), include it as a monospaced text box with `SF_LIGHT` background. Otherwise, summarize what the code does in a bullet point.

### 11. Module / Feature Cards

**How to detect:**
```python
modules = soup.find_all(class_=re.compile(r"module-card|feature-card"))
```

**Content to extract:**
- Icon/emoji → card icon
- `h3` → card title
- `p` → card description

**Handling:** If 3-4 cards, use Executive Summary layout. If more, group related cards into Detail Slides with bullets.

## Slide Ordering

Build slides in this order:

1. **Title Slide** (from header) — always first
2. **Problem/Intro Slide** (from first narrative section) — always second
3. **Executive Summary** (from pillar grid) — if pillars exist, always third
4. **Detail Slides** (from feature sections / individual pillars) — middle
5. **Stats Slide** (from stat cards) — if stats exist
6. **Comparison Table** (from table) — near the end
7. **CTA/Closing Slide** (from CTA + footer) — always last

## Consolidation Rules

- If the HTML has fewer than 4 content sections, skip the Executive Summary and go straight to detail slides.
- If a callout appears right after a detail section, fold it into that slide as a highlighted box rather than creating a separate slide.
- If there are more than 6 detail sections, consolidate related ones (combine bullets) to keep the deck under 10 slides.
- Code blocks should almost never get their own slide.
- Stat cards with fewer than 3 items can be folded into an adjacent slide.
