# HTML to PowerPoint Conversion

Convert Snowflake-branded HTML documents (generated by the `snowflake-brand-guidelines` skill) into professional widescreen PowerPoint presentations using `python-pptx`.

## Prerequisites

- Python 3.9+
- `python-pptx` (`pip install python-pptx`)
- `beautifulsoup4` (`pip install beautifulsoup4`)
- Official Snowflake white logo PNG at `~/<project>/snowflake_logo_white.png`
  - Source: extract `Snowflake Logo/Digital/PNG/snowflake-logo-reverse-rgb@2x.png` from `https://www.snowflake.com/wp-content/uploads/2021/04/Snowflake-Logo-Package.zip`
  - Dimensions: 1221x279px, RGBA, transparent background
  - If the logo PNG is not found, download the ZIP and extract it before proceeding

## Brand Constants

Every generated script MUST begin with these constants:

```python
from pptx import Presentation
from pptx.util import Inches, Pt, Emu
from pptx.dml.color import RGBColor
from pptx.enum.text import PP_ALIGN, MSO_ANCHOR
from pptx.enum.shapes import MSO_SHAPE

# Snowflake brand colors
SF_BLUE = RGBColor(0x29, 0xB5, 0xE8)       # Primary brand blue
SF_MID_BLUE = RGBColor(0x11, 0x56, 0x7F)   # Headers, dark accents
SF_DARK = RGBColor(0x00, 0x00, 0x00)        # Midnight black
SF_STAR_BLUE = RGBColor(0x71, 0xD3, 0xDC)  # Secondary accent
SF_ORANGE = RGBColor(0xFF, 0x9F, 0x36)      # Attention/warnings
SF_GRAY = RGBColor(0x8A, 0x99, 0x9E)        # Muted text
SF_LIGHT = RGBColor(0xF0, 0xF8, 0xFF)       # Light background
SF_BODY = RGBColor(0x5B, 0x5B, 0x5B)        # Body text
SF_ICEBERG = RGBColor(0x00, 0x35, 0x45)     # Deep dark background
SF_WHITE = RGBColor(0xFF, 0xFF, 0xFF)        # White
SF_GREEN = RGBColor(0x2E, 0xCC, 0x71)       # Success/check marks

# Widescreen 16:9
SLIDE_W = Inches(13.333)
SLIDE_H = Inches(7.5)
```

## Helper Functions

Include these helper functions in every generated script. They are the building blocks for all slide types.

```python
def add_background(slide, color):
    """Set solid background color on a slide."""
    bg = slide.background
    fill = bg.fill
    fill.solid()
    fill.fore_color.rgb = color


def add_rect(slide, left, top, width, height, fill_color, line_color=None):
    """Add a filled rectangle shape."""
    shape = slide.shapes.add_shape(MSO_SHAPE.RECTANGLE, left, top, width, height)
    shape.fill.solid()
    shape.fill.fore_color.rgb = fill_color
    if line_color:
        shape.line.color.rgb = line_color
    else:
        shape.line.fill.background()
    return shape


def add_rounded_rect(slide, left, top, width, height, fill_color):
    """Add a rounded rectangle."""
    shape = slide.shapes.add_shape(MSO_SHAPE.ROUNDED_RECTANGLE, left, top, width, height)
    shape.fill.solid()
    shape.fill.fore_color.rgb = fill_color
    shape.line.fill.background()
    return shape


def add_text_box(slide, left, top, width, height, text, font_size=18,
                 color=SF_BODY, bold=False, font_name="Calibri",
                 alignment=PP_ALIGN.LEFT, anchor=MSO_ANCHOR.TOP):
    """Add a text box with a single run of text."""
    txBox = slide.shapes.add_textbox(left, top, width, height)
    txBox.text_frame.word_wrap = True
    txBox.text_frame.vertical_anchor = anchor
    p = txBox.text_frame.paragraphs[0]
    p.alignment = alignment
    run = p.add_run()
    run.text = text
    run.font.size = Pt(font_size)
    run.font.color.rgb = color
    run.font.bold = bold
    run.font.name = font_name
    return txBox


def add_bullet_list(slide, left, top, width, height, items, font_size=14,
                    color=SF_BODY, bullet_color=SF_BLUE):
    """Add a text box with bullet points."""
    txBox = slide.shapes.add_textbox(left, top, width, height)
    tf = txBox.text_frame
    tf.word_wrap = True
    for i, item in enumerate(items):
        if i == 0:
            p = tf.paragraphs[0]
        else:
            p = tf.add_paragraph()
        p.space_after = Pt(6)
        p.space_before = Pt(2)
        p.level = 0
        bullet_run = p.add_run()
        bullet_run.text = "\u2022  "
        bullet_run.font.size = Pt(font_size)
        bullet_run.font.color.rgb = bullet_color
        bullet_run.font.bold = True
        bullet_run.font.name = "Calibri"
        text_run = p.add_run()
        text_run.text = item
        text_run.font.size = Pt(font_size)
        text_run.font.color.rgb = color
        text_run.font.name = "Calibri"
    return txBox
```

## Workflow

### Step 1: Read and Analyze the Source HTML

Read the HTML file. Identify its structure using CSS class names and semantic elements. Load `references/html-section-map.md` for the full mapping of brand HTML classes to slide types.

Key things to note:
- The document title (from `<h1>` in `.header` or `<title>`)
- The subtitle (from `.header p` or `.header-subtitle`)
- Section types present (see Section Mapping below)
- Number of sections (to estimate slide count -- aim for 6-10 slides)
- Any comparison tables, stat cards, or pillar grids

### Step 2: Ensure Dependencies

```bash
pip install python-pptx beautifulsoup4
```

Verify the logo PNG exists. If not, download and extract:

```bash
curl -L -o /tmp/snowflake-logos.zip "https://www.snowflake.com/wp-content/uploads/2021/04/Snowflake-Logo-Package.zip"
unzip -o /tmp/snowflake-logos.zip -d /tmp/snowflake-logos
cp "/tmp/snowflake-logos/Snowflake Logo/Digital/PNG/snowflake-logo-reverse-rgb@2x.png" "<project_dir>/snowflake_logo_white.png"
```

### Step 3: Plan the Slide Deck

Map HTML sections to slides. Use this decision logic:

| HTML Section | Slide Type | Notes |
|---|---|---|
| `.header` (gradient, h1, subtitle) | **Title Slide** | Dark header band + colorstack title + subtitle + logo |
| First `<section>` with intro text | **Problem/Intro Slide** | Big quote + body text with left accent line |
| `.pillars` grid (2-4 cards) | **Executive Summary Slide** | Multi-column cards with colored headers |
| Individual `.pillar` or feature `<section>` with bullets | **Detail Slide** | Icon circle + heading + bullet list |
| `table` with comparison data | **Comparison Table Slide** | Styled table with check/cross/partial indicators |
| `.callout` box | Fold into adjacent slide or **Callout Slide** | Rounded rect with accent color |
| `.stat-card` grid | **Stats Slide** | Large numbers with labels in cards |
| `.cta-bar` or closing section | **CTA/Closing Slide** | Dark background + big text + button + logo + copyright |
| `.footer` | Fold into CTA slide | Copyright text only |

**Guidelines:**
- Aim for 6-10 slides total. Fewer is better for exec audiences.
- Combine related small sections into one slide rather than creating many sparse slides.
- If the HTML has 3+ pillar/feature sections, create BOTH an executive summary (all pillars on one slide) AND individual detail slides.
- Always start with a Title Slide and end with a CTA/Closing Slide.
- Add the white logo to the Title Slide (top-right) and the CTA Slide (top-right).

### Step 4: Generate the Python Script

Write a Python script that builds the PPTX. The script should:

1. **Initialize** the presentation with widescreen dimensions
2. **Include** all brand constants and helper functions from above
3. **Build slides** in order using the patterns from `references/slide-types.md`
4. **Extract text** from the HTML -- use the actual content from the source, not placeholder text
5. **Save** to the output path

Load `references/slide-types.md` for the exact python-pptx code pattern for each slide type.

**Script structure:**

```python
"""Generate <title> PowerPoint from HTML content."""

from pptx import Presentation
# ... all imports and constants ...
# ... all helper functions ...

prs = Presentation()
prs.slide_width = Inches(13.333)
prs.slide_height = Inches(7.5)

# SLIDE 1: TITLE
# ... title slide code ...

# SLIDE 2: PROBLEM/INTRO
# ... intro slide code ...

# SLIDE 3: EXECUTIVE SUMMARY (if pillars present)
# ... multi-column card slide ...

# SLIDES 4-N: DETAIL SLIDES
# ... one per major section ...

# SLIDE N+1: COMPARISON TABLE (if table present)
# ... table slide code ...

# SLIDE N+2: CTA/CLOSING
# ... closing slide code ...

output_path = "/path/to/output.pptx"
prs.save(output_path)
print(f"Saved: {output_path}")
```

**Default output naming:** Same basename as the HTML file with `.pptx` extension, in the same directory.

### Step 5: Run and Verify

Execute the script:

```bash
python create_slides.py
```

After generation:
1. Confirm the file was created and note its size
2. Report the slide count and structure to the user
3. Open the file if on macOS: `open output.pptx`

### Step 6: Handle Errors

**`ModuleNotFoundError: No module named 'pptx'`:**
- Run `pip install python-pptx` (note: the package name is `python-pptx`, not `pptx`)

**Logo not found:**
- Download from the Snowflake Logo Package ZIP (see Step 2)
- The white logo PNG is at: `Snowflake Logo/Digital/PNG/snowflake-logo-reverse-rgb@2x.png`

**Text overflow / truncation:**
- Reduce font size or split content across multiple slides
- For bullet lists, limit to 4-5 items per slide

**Table too wide:**
- Reduce column count or abbreviate text
- Use `Pt(11)` or `Pt(12)` font size for dense tables

**Emoji/Unicode rendering:**
- python-pptx supports Unicode, but some emoji may not render in all PowerPoint versions
- Prefer simple Unicode symbols: `\u2713` (check), `\u2717` (cross), `\u26A1` (lightning), `\U0001F517` (link), `\U0001F6E1` (shield), `\u2192` (arrow)

## Section Mapping Reference

The `snowflake-brand-guidelines` skill produces HTML with consistent patterns. Here is how to identify each section type:

| CSS Selector / Pattern | Content Type | How to Identify |
|---|---|---|
| `.header` with `linear-gradient` | Title + subtitle | Always first element, contains `<h1>` and optional `<p>` |
| `<section>` with long `<p>` text, no cards | Intro/problem statement | Usually second section, narrative text |
| `.pillars` or `.summary-grid` | Pillar cards (2-4) | Grid of cards with `border-top: 4px solid` |
| `.pillar` or card with `<h3>` + `<ul>` | Feature detail | Individual card within a grid |
| `<table>` with `<thead>` + `<tbody>` | Comparison table | Contains `\u2713`, `\u2717`, `~` indicators |
| `.stat-card` or `.metric` | Statistics | Large number + label pairs |
| `.callout` | Callout/highlight box | Colored left border or background |
| `.cta-bar` or section with button/link | Call to action | Contains "Get Started", "Learn More", etc. |
| `.footer` with `\u00A9` | Copyright footer | Always last element |
| `<section>` with `<h2>` + `<ul>` | Generic content | Standard heading + bullets |
| `.module-card` | Module/feature card | Card with icon + description |
| `<pre>` or `<code>` | Code block | Skip or summarize in slides |
| `.agenda` or numbered list | Agenda/outline | Numbered items |
